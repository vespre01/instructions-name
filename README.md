# Инструкция по созданию локального репозитория (проекта)

1.Создать папку проекта по пути ~/dev/ на UI или через консоль

2.В консоли перейти в папку проекта из 1 шага и прописать команду 

```

git init

```

# Инструкция по связи удаленного репозитория с локальным

Предусловия: создан локальный репозиторий (git init)

1.Зайти на GitHub (https://github.com/username)
2.Зайти во вкладку repositories и создать новый репозиторий на github
3.Связать удаленный репозиторий с локальным: скопировать git@github.com:username/projectname.git и перейти в консоль и прописать git remote add origin git@github.com:username/projectname.git
4.Проверить текущее состояние репозитория, прописав в консоли команду 

```
git status

```
5.Проверить что репозитории связаны, прописав команду в консоли 

```
git remote –v

```


# Инструкция по добавлению изменений в удаленный репозиторий

1.Находясь в рабочей директории, локально добавить файл, прописав в консоли команду 

```
touch

```
2.На UI отредактировать файл и сохранить
3.Добавить файл для отслеживания изменений, прописав через консоль команду

```
git add

```
4.Проверить состояние репозитория, прописав команду 

```
git status

```
5.Сохранить изменения (закоммитить), прописав через консоль команду 

```
git commit -m ‘Тут прописать комментарий что изменилось’

```
6.Запушить изменения в репозиторий, прописав команду 

```
git push -u origin master 

```

(если первый раз в данный репозиторий), дальше достаточно использовать 

```
git push

```
7.Проверить состояние репозитория, прописав команду 

```
git status

```
8.Зайти на GitHub (https://github.com/username) и проверить, что локальные изменения продублировались в удаленный репозиторий
9.Посмотреть историю коммитов, прописав команду

```
git log

```


# Ключевые понятия Git

**Хэш** - основной идентификатор коммита и позволяет узнать его автора, дату и содержимое закоммиченных файлов.
Git преобразует информацию о коммитах с помощью алгоритма SHA-1 и для каждого из них рассчитывает уникальный идентификатор — хеш.
Все хеши, а также таблицу соответствий "хеш → информация о коммите" Git хранит в папке .git.
